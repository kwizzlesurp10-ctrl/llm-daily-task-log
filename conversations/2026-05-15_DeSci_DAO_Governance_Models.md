## VitaDAO Model (Longevity Focus)
- Token-weighted governance: VITA ERC-20 token. 1 token = 1 vote.
- Proposal types: governance, project funding, IP, data monetization.
- Inspired by MolochDAO + LAO. Gnosis Safe multisig for execution.
- Community (scientists, researchers, Pfizer, crypto figures) votes on research funding and IP.
- Full token holder control. No CEO/board. Proposals from members.
- Staking + governance mechanisms.

## bio.xyz / BioDAO Model
- BIO token curation: Stake BIO to vote/curate which BioDAOs join ecosystem. More stake = more power.
- Each BioDAO: Own governance. Token holders vote on project funding, strategy, treasury allocation.
- Hybrid: Expert/working group review proposals for merit before community vote.
- Meta-governance: BIO holders govern multiple BioDAOs + IP via treasury.
- IP Tokens (IPTs): ERC-20 for governance over specific scientific IP, licensing, revenue share.
- Snapshot for proposals. Focus on curation, funding, liquidity, rewards.

## Common DeSci DAO Governance Patterns
- Token-weighted voting (dominant): Simple, aligns with stake. Pitfalls: whale dominance.
- Quadratic voting elements: Sqrt(token) power for fairer representation (Gitcoin-inspired, some Snapshot strategies). Reduces concentration.
- Progressive decentralization: Start multisig/curated, evolve to full on-chain token vote.
- Delegation + timelocks: For participation and security.
- Hybrid expert + community: Scientific review before token vote.
- On-chain execution: Smart contracts + multisig for treasury/IP decisions.
